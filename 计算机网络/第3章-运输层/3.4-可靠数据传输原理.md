# 可靠数据传输原理

- **构建可靠数据传输协议**
  - **rdt1.0**: 经完全可靠信道的可靠数据传输
  - **rdt2.0**: 经具有比特差错信道的可靠数据传输 (ARQ: 自动重传请求协议)
    - 差错检测
    - 确认 (ACK)
    - 否认 (NAK)
  - **rdt3.0**: 经具有比特差错的丢包信道的可靠数据传输
    - 引入定时器和超时重传
- **流水线可靠数据传输协议**
  - 解决停等协议 (Stop-and-Wait) 效率低的问题
  - **回退N步 (Go-Back-N, GBN)**
    - 发送方维护一个发送窗口
    - 接收方只按序接收
  - **选择重传 (Selective Repeat, SR)**
    - 发送方和接收方都维护一个窗口
    - 接收方可以缓存失序的分组
    - 发送方只重传丢失的分组
