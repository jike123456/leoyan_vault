# TCP 拥塞控制

- **拥塞的原因与代价**
  - 多条连接在路由器上竞争有限的带宽
  - 表现：分组丢失、时延增加
- **拥塞控制的方法**
  - 端到端拥塞控制 (TCP采用)
  - 网络辅助的拥塞控制
- **TCP 拥塞控制算法**
  - **慢启动 (Slow Start)**
    - 拥塞窗口 (cwnd) 指数增长
  - **拥塞避免 (Congestion Avoidance)**
    - cwnd 线性增长
  - **快速重传 (Fast Retransmit)**
    - 收到3个冗余ACK时，不等超时就重传
  - **快速恢复 (Fast Recovery)**
- **TCP Tahoe 和 TCP Reno**
  - 两种经典的TCP拥塞控制算法实现
- **公平性**
  - TCP如何在多个连接之间公平地共享带宽
