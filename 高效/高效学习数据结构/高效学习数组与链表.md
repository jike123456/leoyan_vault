# 高效学习数组与链表 (线性表基础)

## 一、 宏观把控：内存的双重人格
**“黄金圈”思考**
- **What (本质)**：
    - **数组 (Array)**：**连续**的内存块。就像电影院连号的座位。
    - **链表 (Linked List)**：**离散**的内存块，靠“指针”串联。就像寻宝游戏，线索纸条藏在不同房间，找到第一张才能去下一个房间。
- **Why (核心权衡)**：
    - 数组：**查**找快 (O(1))，**增删**慢 (O(N)) —— *适合读多写少*。
    - 链表：**增删**快 (O(1))，**查**找慢 (O(N)) —— *适合频繁插入删除*。
- **How (操作)**：
    - 数组用索引 `arr[i]` 访问。
    - 链表用 `head.next` 顺藤摸瓜。

### 1. ==虚拟头结点 (Dummy Node)== —— 必用！
- **痛点**：处理头节点（head）和其他节点逻辑不一样，比如要删除 head，还得特判。
- **解法**：人为在 head 前面加个 `dummy` 节点。
- **模板**：
  ```java
  ListNode dummy = new ListNode(-1);
  dummy.next = head;
  // ... 操作 ...
  return dummy.next; // 返回真正的头
  ```

### 2. 双指针 (Two Pointers) —— 数组/链表通杀
- **快慢指针 (Fast & Slow)**：
    - **场景**：链表判环、找链表中点。
    - **原理**：==快指针一次走两步，慢指针一次走一步。如果有环，快的肯定能套圈追上慢的==。
- **左右指针 (Left & Right)**：
    - **场景**：数组反转、二分查找。
    - **原理**：==一头一尾向中间逼近==。

### 3. 反转链表 (Reverse) —— 基本功
- **口诀**：保存 `next`，改变指向，整体后移。
- **必须默写**：这是很多大厂面试的第一题，写不出来直接挂。

## 三、 针对您的“特效药”：画图与模拟
**链表是指针的游戏，脑子想不清楚，必须画！**

1.  **物理模拟 (指针断链)**：
    - 画几个圆圈代表节点，用箭头代表 `next` 指针。
    - **删除节点**：想象“过河拆桥”。把 A->B->C 中的 B 删掉，就是把 A 的箭头直接拉长，指到 C 身上（`A.next = A.next.next`），B 就悬空了。
    - **插入节点**：想象“接水管”。先把新管子的一头接上后面，再把前面的管子接到新管子上。（**顺序千万不能反，反了就断链了！**）
2.  **代码模拟**：
    - 在纸上写出 `prev`, `curr`, `next` 三个变量，每执行一行代码，就更新一下它们指向谁。
