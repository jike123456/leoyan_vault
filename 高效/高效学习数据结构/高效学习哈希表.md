# 高效学习哈希表 (Hash Table)

## 一、 宏观把控：空间换时间的魔法
**“黄金圈”思考**
- **What (本质)**：一个超级数组。通过 **哈希函数 (Hash Function)** 算出索引，直接定位。
- **Why (核心优势)**：**O(1)** 的查找速度。只要题目里出现“查找”、“统计次数”、“判断存在”，第一时间想哈希表。
- **How (代价)**：占内存。这就是典型的 **空间换时间**。

## 二、 核心技巧三板斧

### 1. Map vs Set (分清谁是谁)
- **Set (集合)**：
    - **只存 Key**。
    - **用途**：去重、判断某元素是否存在。
    - *例子*：判断黑名单用户。
- **Map (字典/映射)**：
    - **存 Key-Value 对**。
    - **用途**：不仅要知道“有没有”，还要知道“有多少/在哪儿”。
    - *例子*：统计每个单词出现的次数（Key=单词, Value=次数）。

### 2. 计数器 (Counter)
- **场景**：统计字符/数字出现频率。
- **技巧**：
    - 如果字符集很小（比如只有小写字母），可以用 `int[26]` 数组代替 Map，速度更快。
    - 否则用 `HashMap<Key, Integer>`。

### 3. 记录索引
- **场景**：两数之和。
- **技巧**：Key 存数值，Value 存数组下标。这样找到目标值时，能立刻通过 Value 拿回它的位置。

## 三、 针对您的“特效药”：查字典模拟

1.  **哈希函数 = 拼音索引**：
    - 你要查 "Zhong" 这个字。
    - 你不会从第一页翻到第 500 页（这是链表/数组遍历 O(N)）。
    - 你查拼音表 "Z"，直接跳转到第 400 页（这是哈希 O(1)）。
2.  **哈希碰撞 = 同音字**：
    - 如果 "Zhang" 和 "Zhong" 算出来的页码一样怎么办？
    - 既然撞了，就在那一页拉一条链子（链地址法），把两个字都挂在那。

## 五、 费曼技巧自测
> “如果题目限制空间复杂度为 O(1)，我还能用哈希表吗？为什么？”
> *(答案：不能。哈希表本质就是消耗额外的 O(N) 空间来存储映射表。如果限制 O(1) 空间，通常要考虑双指针或者位运算。)*
