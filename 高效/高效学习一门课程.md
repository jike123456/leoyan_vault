# 高效学习一门课程 (通用方法论)

## 一、 核心原则：主动学习 (Active Recall)
- **被动学习**：听课、看书、划线（留存率 < 30%）。
- **主动学习**：做项目、教授他人、自我测试（留存率 > 70%）。
- **结论**：不要只是“看”视频，要动手“做”。

## 二、 学习流程 (SOP)

### 1. 预习与宏观把控 (Overview)
- **动作**：在开始深入细节前，先看目录、大纲、思维导图。
- **目的**：建立知识的“书架”。你需要知道现在学的知识点属于“哪一个柜子”。
- **提问**：这门课解决什么核心问题？（例如：操作系统解决了计算机资源管理的问题）。

### 2. 费曼技巧 (The Feynman Technique) —— 检验是否真懂
- **步骤**：
    1.  拿出一张白纸，写下你要学的概念。
    2.  **想象你在给一个 12 岁的小学生讲解这个概念**。
    3.  如果你卡住了，或者用了很多专业术语（黑话）来解释，说明你没真懂。
    4.  回到书本/视频，重新学习卡住的部分，直到能用大白话讲清楚。

### 3. 黄金圈法则 (Why - How - What)
每学一个新知识点（例如 C++ 的指针），按顺序问自己：
- **Why**：为什么需要指针？（为了直接操作内存，提高效率）。
- **How**：怎么用？（语法是什么，*p, &a）。
- **What**：它本质是什么？（一个存储地址的变量）。

### 4. 艾宾浩斯复习 (Spaced Repetition)
- **规律**：遗忘是指数级的。
- **节点**：学完后的 **1小时、1天、1周、1个月** 是关键复习点。
- **操作**：利用碎片时间快速回顾笔记，或者用 Anki 制作抽认卡。

## 三、 针对“视频课程”的特别建议
1.  **倍速播放**：熟悉的内容 1.5x/2.0x，难点正常速或回放。
2.  **不要暂停抄代码**：看懂逻辑 -> 暂停 -> 自己试着写 -> 报错 -> 对比。**抄代码没有过脑子，是无效努力**。
3.  **Project-Based Learning (PBL)**：
    - 学完一章，立刻做一个小 Demo。
    - 哪怕是学英语，也可以设定一个 Project（例如：录制一段 2 分钟的全英文自我介绍视频）。

## 四、 专注力管理
- **番茄工作法**：25 分钟专注 + 5 分钟休息。
- **手机隔离**：学习时把手机扔到另一个房间。
